# PLO ポストフロップ Equity

PLO（ポットリミット・オマハ）のポストフロップにおける **エクイティ計算** と、
**ランナウト・エクイティ分布グラフ** を表示するスマホ向けWebアプリです。
書籍『PLO From Scratch』Part 3 のフロップ・エクイティ分布（FED）の考え方を、
任意の手札・フロップ・相手レンジに対して再現できます。

ビルド不要・依存パッケージなしの素のHTML/CSS/JS（ESモジュール）で動きます。

---

## できること

- **エクイティ計算タブ**
  自分の4枚 ＋ フロップ（3枚、任意でターン）＋ 相手レンジ を入力すると、
  モンテカルロ法で **勝ち/分け/負け** と総合エクイティを計算します。
- **分布グラフタブ**
  各ランナウト（ターン＋リバー）でのエクイティを高い順に並べた曲線を描きます。
  曲線をなぞると「上位◯%のランナウトで最低◯%のエクイティ」が読め、
  面積（平均線）がそのフロップでの総合エクイティに対応します。
- **相手レンジ記法**（カンマ区切りで和集合）
  | 記法 | 意味 |
  |---|---|
  | `AA**` | エースちょうど2枚＋任意2枚（6,768通り） |
  | `BBBB` / `BBB9` | ブロードウェイ(T〜A)4枚 / 3枚＋9 |
  | `AKQJ` | 各ランク1枚（スート任意） |
  | `AsKsQhJh` | 具体的な4枚 |
  | `AA**,KK**` | 複数レンジの和 |

## スマホで使う（推奨：静的ホスティング）

このアプリはESモジュール＋Web Workerを使うため、**`file://` で直接開くのではなく
HTTP(S) で配信**してください。スマホで使う場合は以下のいずれかが簡単です。

### 方法A: GitHub Pages（無料・URLでどこからでも）
1. `ploequity/` の中身をGitHubリポジトリにpush
2. リポジトリの Settings → Pages で公開ブランチ/フォルダを指定
3. 発行されたURLをスマホで開く →「ホーム画面に追加」でアプリのように使えます
   （PWA対応。オフラインでも起動します）

### 方法B: 同一Wi-Fi内のPCから配信
PCで以下を実行し、スマホのブラウザで `http://<PCのIPアドレス>:8123/` を開きます。
```bash
cd ploequity
python -m http.server 8123
```

### 方法C: 任意の静的ホスティング
Netlify / Vercel / Cloudflare Pages などに `ploequity/` をそのまま置くだけで動きます。

> オフライン用のサービスワーカー（`sw.js`）は **network-first**（オンライン時は常に最新、
> オフライン時のみキャッシュ）です。更新が反映されない問題は起きません。

## ローカルで動かす・検証する

```bash
cd ploequity
python -m http.server 8123      # http://127.0.0.1:8123/ を開く

# エンジンの正しさを検証（書籍の実エクイティ値23件を再現）
node test/golden.test.mjs
```

## ファイル構成

```
ploequity/
├── index.html              画面構造
├── styles.css              モバイルファーストのスタイル
├── manifest.webmanifest    PWA（ホーム画面追加）
├── sw.js                   サービスワーカー（network-first・任意）
├── icon.svg                アプリアイコン
├── src/
│   ├── cards.js            カードのパース/整形（純粋ロジック）
│   ├── evaluator.js        5枚役評価・オマハ式ベスト（純粋ロジック）
│   ├── range.js            レンジ記法の展開（純粋ロジック）
│   ├── equity.js           モンテカルロ・分布計算（純粋ロジック）
│   ├── worker.js           重い計算を分離するモジュールワーカー
│   ├── chart.js            SVG描画（分布曲線・勝率バー）
│   ├── ui.js               UI構築・状態管理・localStorage
│   └── app.js              ワーカー連携・エントリポイント
└── test/
    └── golden.test.mjs     書籍の実数値による回帰テスト
```

純粋ロジック（cards/evaluator/range/equity）はNodeでもそのまま実行でき、テスト可能です。
UI/描画（ui/chart）はDOMに依存します。

---

## 精度について

- 計算はモンテカルロ法のため、実行ごとに小さな誤差（±0.1〜0.5%程度）があります。
  精度セレクタ（高速/標準/高精度）で試行回数を調整できます。
- 書籍掲載の代表的な局面では、ProPokerToolsの値を ±0.3% 程度で再現します
  （`test/golden.test.mjs` 参照）。
  Pages deploy retry.
